2.2 Data and study design

El conjunto de datos usados en este informe procede de muestras de sangre de adulto sano y con mieloma mltiple del estudio con PMID:24816239 para detectar genes diferencialmente expresados mediante el uso de la tecnologa RNAseq (plataforma Illumina Genome Analyzer II) [1]. Los datos completos originales se encuentran disponibles en el repositorio Gene expression omnibus (GEO) bajo el accession number GSE47552. No obstante, en este estudio slo se analizan datos de pacientes sanos vs con mieloma mltiple.

En cuanto al tipo de estudio, el anlisis aqu presentado puede clasificarse como un estudio comparativo (Class comparison), ya que se comparan muestras de pacientes sanos con mieloma mltiple.

En lo que respecta al diseo experimental, miramos los siguientes puntos:

  • Fuentes de variabilidad: Quizs la obtencin de muestras pudiera haber sido algo hetereognea entre muestras, lo que podra introducir variabilidad en los resultados, pero el hecho de tener rplicas biolgicas podra minimizar su impacto. Si a partir de aqu las muestras se obtienen o procesan de manera homognea (en paralelo o con un bloqueo adecuado), la variabilidad no debida a las diferentes condiciones se pueda minimizar. Valoraremos si las fechas de ejecucin de los microarrasys dan lugar a un batch factor que pueda influenciar los resultados del anlisis.

  • No se han empleado rplicas tcnicas.

  • Rplicas biolgicas: Cinco de pacientes sanos y 41 con mieloma mltiple.

  • Aleatorizacin: No procede en este caso al estar comparando.

library(ggplot2)
library(ggrepel)
plotPCA3 <- function (datos, labels, factor, title, scale,colores, size = 1.5, glineas = 0.25) {
  data <- prcomp(t(datos),scale=scale)
  
  # plot adjustments
  dataDf <- data.frame(data$x)
  Group <- factor
  loads <- round(data$sdev^2/sum(data$sdev^2)*100,1)
  
  # main plot
  p1 <- ggplot(dataDf,aes(x=PC1, y=PC2)) + theme_classic() +
    geom_hline(yintercept = 0, color = "gray70") +
    geom_vline(xintercept = 0, color = "gray70") +
    geom_point(aes(color = Group), alpha = 0.55, size = 3) +
    coord_cartesian(xlim = c(min(data$x[,1])-5,max(data$x[,1])+5)) +
    scale_fill_discrete(name = "Group")
  
  # avoiding labels superposition
  p1 + geom_text_repel(aes(y = PC2 + 0.25, label = labels),segment.size = 0.25, size = size) + 
    labs(x = c(paste("PC1",loads[1],"%")),y=c(paste("PC2",loads[2],"%"))) + 
    ggtitle(paste("Principal Component Analysis para",title,sep=" ")) + 
    theme(plot.title = element_text(hjust = 0.5)) +
    scale_color_manual(values=colores)
}

3.4 Batch detection

Puede ser que los microarrays de este estudio se hayan visto afectados por variables no biolgicas tales como el uso de reactivos de diferentes lotes, diferentes tcnicos llevando a cabo el protocolo, o la ms comn, la fecha de procesado. El hecho de que en el estudio no se mencione la existencia de bloqueo puede hacer pensar que las muestras se hayan procesado de manera homognea, bien todas el mismo da, o bien con muestras de cada grupo includas en cada da de procesamiento de muestras. Sin embargo, a partir de la funcin get.celfile.dates() aplicada a los archivos .CEL podemos ver que todos los anlisis con microarrays no se realizaron el mismo da, sino que se distribuyeron en diferentese fechas de manera no totalmente homognea, lo cual quizs haya podido introducir variabilidad en los datos obtenidos y produciendo el denominado batch effect:

3.6 Filtering of genes with lowest variability

Con el objetivo de aumentar la potencia de nuestro anlisis y por tanto reducir el error tipo II (falsos negativos) conviene reducir el nmero de genes a los que les aplicamos el contraste para determinar si estn diferencialmente expresados. Para ello eliminaremos los genes con menos probabilidad de estar expresados (es decir, aqullos que tengan menor variabilidad) mediante el uso de la funcin nsFilter del paquete geneFilter. Tambin filtraremos sondas sin un identificador del gen asociado, para lo cual debemos usar el paquete de anotacin correspondiente al chip usado en este estudio (hugene10sttranscriptcluster.db):

library(genefilter)
library(hugene10sttranscriptcluster.db)
annotation(eset_rma) <- "hugene10sttranscriptcluster.db"
filtered <- nsFilter(eset_rma, 
                     require.entrez = TRUE, remove.dupEntrez = TRUE,
                     var.filter=TRUE, var.func=IQR, var.cutoff=0.75, 
                     filterByQuantile=TRUE, feature.exclude = "^AFFX")

annotatedTopTable <- function(topTab, anotPackage)
{
  topTab <- cbind(PROBEID=rownames(topTab), topTab) # Se aade columna con PROBEIDs
  myProbes <- rownames(topTab)
  
  # Objeto ChipDb a partir de paquete de anotacin:
  thePackage <- eval(parse(text = anotPackage)) 
  
  # Seleccin de columnas de inters del objeto ChipDb:
  geneAnots <- AnnotationDbi::select(thePackage, myProbes, c("SYMBOL", "ENTREZID", "GENENAME")) 
  
  # Fusin de columnas seleccionadas en paso anterior con TopTabs segn PROBEID:
  annotatedTopTab<- merge(x=geneAnots, y=topTab, by.x="PROBEID", by.y="PROBEID") 
return(annotatedTopTab)
}
