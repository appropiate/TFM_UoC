1 Abstract

En este informe se muestran genes diferencialmente expresados en muestras de pacientes adultos con mieloma múltiple, así como los correspondientes procesos biológicos enrriquecidos.

2 Materials and Methods

2.1 Software

El análisis bioinformático presentado en este informe esta basado en el lenguaje R (R version 4.0.3 (2020-10-10)) y el uso de RStudio como interfaz de usuario, donde se han usado los siguientes paquetes tanto del repositorio CRAN como del repositorio de Bioconductor:

2.2 Data and study design

El conjunto de datos usados en este informe procede de muestras de sangre de adulto sano y con mieloma múltiple del estudio con PMID:34133718 para detectar genes diferencialmente expresados mediante el uso de la tecnología RNAseq (plataforma Illumina Genome Analyzer II) [1]. Los datos completos originales se encuentran disponibles en el repositorio Gene expression omnibus (GEO) bajo el accession number GSE175384. No obstante, en este estudio sólo se analizan datos de humanos adultos sanos vs con mieloma múltiple.

En cuanto al tipo de estudio, el análisis aquí presentado puede clasificarse como un estudio comparativo (Class comparison), ya que se comparan muestras de la especie humana de adulto sano y con mieloma múltiple.

El método de análisis de datos usado en este informe se basa en el paquete DESeq2 y sigue un pipeline similar al usado en el informe RNASeqBiocPipeline.html, al cual se accede a partir del enlace:

https://github.com/ASPteaching/Omics_data_analysis-Case_study_2-RNA-seq

Sin embargo, se introducen algunas variaciones como, por ejemplo, el análisis de enrriquecimiento, el cual no está incluído en dicha plantilla.

3.2 Quality control of raw data

Debido a que el análisis se lleva a cabo a partir del ‘count table,’ asumimos que éstos son los datos crudos y por tanto aplicamos el control de calidad y demás análisis a los mismos.

3.2.1 Data pre-filtering

Muchas de las filas de dds corresponden a genes con expresión génica indetectable en todas las muestras y no contienen conteos (es decir, contienen solamente ceros), luego no son muy informativas. Por ello, procedemos a filtrar (elimininar) dichas filas para obtener un ´DESeqDataSet´ más pequeño que permita ejecutar las funciones aplicadas a este objeto con más rapidez. Para el filtraje usaremos en este caso el criterio más básico: Eliminar las filas de dds sin conteos o que bien contengan un único conteo teniendo en cuenta todas las columnas (muestras):

# Número de genes inicial:
(a <- nrow(dds))
[1] 48495
# Aplicamos prefiltraje:
dds <- dds[rowSums(counts(dds)) > 1, ]

# Genes en dds después del filtraje:
(b <- nrow(dds))
[1] 38403

Vemos que aproximadamente nos quedamos con un 79% del número inicial de genes contenidos en dds.

3.2.2 Normalization

Como se menciona en la plantilla RNASeqBiocPipeline.html, se recominenda no normalizar los datos para identificar genes diferencialmente expresados mediante la función DESeq(). Sin embargo, dichos datos sí que se normalizan para visualizarlos y obtener una idea de su calidad. Para ello, DESeq2 ofrece dos tipos de transformaciones para datos de conteo que estabilizan la varianza sobre la media: variance stabilizing transformation (VST) [2], la cual se implementa con la función vst(), y regularized-logarithm (rlog) [3], implementada con la función rlog. Como también se recomienda en RNASeqBiocPipeline.html, en este caso elegimos la transformación VST para visualizar datos en gráficos posteriores ya que esta función se computa más rápidamente, es menos sensible que rlog a outliers con muchos conteos, y además tenemos en nuestro estudio muestras con muchos conteos (n>30).

3.3 Identification of differentially expressed genes (DEGs)

Para identificar los DEGs usamos primeramente la función DESseq(), para la cual se han de usar los datos crudos (objeto dds que creamos anteriormente). Con ello se genera un objeto DESseqDataSet , el cual se usa a su vez como argumento en la función results() junto con el contraste que deseamos para obtener una tabla con los resultados estadísticos y logFC para cada gen:

dds <- DESeq(dds, parallel =F)

3.5.2 MA plot

El MA plot proporciona una descripción general útil para la distribución de los coeficientes estimados en el modelo, es decir, las comparaciones de interés, en todos los genes. Un MA plot con un alto número de puntos de datos por encima del umbral en el eje de ordenadas indicaría un mayor número de genes sobreexpresados, mientras que más puntos por debajo de -1 indicaría un número mayor de genes infraexpresados [4].

Antes de realizar el gráfico MA, usamos la función lfcShrink para reducir los log2FC para la comparación de muestras sanas vs mieloma múltiple. Entre otros métodos, DESeq2 ofrece el método apeglm para reducir los coeficientes, el cual es bueno para reducir las estimaciones de LFC que se considerarían ruido al mismo tiempo que proporciona estimaciones de LFC de bajo sesgo para diferencias verdaderamente grandes [5]. Para usar apeglm, especificamos un coeficiente a encoger del modelo entre los que aparecen en resultsNames (dds). En nuestro caso sólo tenemos el coeficiente correspondiente a la comparación sano vs mieloma múltiple:

resultsNames(dds)
[1] "Intercept"                                               
[2] "Sample_source_name_ch1_Multiple_Myeloma_vs_Healthy_Adult"
MAdata <- lfcShrink(dds, coef="Sample_source_name_ch1_Multiple_Myeloma_vs_Healthy_Adult", type="apeglm")
plotMA(MAdata, main = 'MA plot para comparación Healthy vs MM', ylim = c(-10,10)) 
topGene <- rownames(MAdata)[which.min(MAdata$padj)]
with(MAdata[topGene, ], {
  points(baseMean, log2FoldChange, col="dodgerblue", cex=2, lwd=2)
  text(baseMean, log2FoldChange, topGene, pos=2, col="dodgerblue")
})
