1 Abstract

En este informe se muestran genes diferencialmente expresados en muestras de pacientes adultos con mieloma mltiple, as como los correspondientes procesos biolgicos enrriquecidos.

2.2 Data and study design

El conjunto de datos usados en este informe procede de muestras de sangre de adulto sano y con mieloma mltiple del estudio con PMID:34133718 para detectar genes diferencialmente expresados mediante el uso de la tecnologa RNAseq (plataforma Illumina Genome Analyzer II) [1]. Los datos completos originales se encuentran disponibles en el repositorio Gene expression omnibus (GEO) bajo el accession number GSE175384. No obstante, en este estudio slo se analizan datos de humanos adultos sanos vs con mieloma mltiple.

En cuanto al tipo de estudio, el anlisis aqu presentado puede clasificarse como un estudio comparativo (Class comparison), ya que se comparan muestras de la especie humana de adulto sano y con mieloma mltiple.

El mtodo de anlisis de datos usado en este informe se basa en el paquete DESeq2 y sigue un pipeline similar al usado en el informe RNASeqBiocPipeline.html, al cual se accede a partir del enlace:

https://github.com/ASPteaching/Omics_data_analysis-Case_study_2-RNA-seq

Sin embargo, se introducen algunas variaciones como, por ejemplo, el anlisis de enrriquecimiento, el cual no est includo en dicha plantilla.

3.2.1 Data pre-filtering

Muchas de las filas de dds corresponden a genes con expresin gnica indetectable en todas las muestras y no contienen conteos (es decir, contienen solamente ceros), luego no son muy informativas. Por ello, procedemos a filtrar (elimininar) dichas filas para obtener un DESeqDataSet ms pequeo que permita ejecutar las funciones aplicadas a este objeto con ms rapidez. Para el filtraje usaremos en este caso el criterio ms bsico: Eliminar las filas de dds sin conteos o que bien contengan un nico conteo teniendo en cuenta todas las columnas (muestras):

3.2.2 Normalization

Como se menciona en la plantilla RNASeqBiocPipeline.html, se recominenda no normalizar los datos para identificar genes diferencialmente expresados mediante la funcin DESeq(). Sin embargo, dichos datos s que se normalizan para visualizarlos y obtener una idea de su calidad. Para ello, DESeq2 ofrece dos tipos de transformaciones para datos de conteo que estabilizan la varianza sobre la media: variance stabilizing transformation (VST) [2], la cual se implementa con la funcin vst(), y regularized-logarithm (rlog) [3], implementada con la funcin rlog. Como tambin se recomienda en RNASeqBiocPipeline.html, en este caso elegimos la transformacin VST para visualizar datos en grficos posteriores ya que esta funcin se computa ms rpidamente, es menos sensible que rlog a outliers con muchos conteos, y adems tenemos en nuestro estudio muestras con muchos conteos (n>30).

3.3 Identification of differentially expressed genes (DEGs)

Para identificar los DEGs usamos primeramente la funcin DESseq(), para la cual se han de usar los datos crudos (objeto dds que creamos anteriormente). Con ello se genera un objeto DESseqDataSet , el cual se usa a su vez como argumento en la funcin results() junto con el contraste que deseamos para obtener una tabla con los resultados estadsticos y logFC para cada gen:

3.5.2 MA plot

El MA plot proporciona una descripcin general til para la distribucin de los coeficientes estimados en el modelo, es decir, las comparaciones de inters, en todos los genes. Un MA plot con un alto nmero de puntos de datos por encima del umbral en el eje de ordenadas indicara un mayor nmero de genes sobreexpresados, mientras que ms puntos por debajo de -1 indicara un nmero mayor de genes infraexpresados [4].

Antes de realizar el grfico MA, usamos la funcin lfcShrink para reducir los log2FC para la comparacin de muestras sanas vs mieloma mltiple. Entre otros mtodos, DESeq2 ofrece el mtodo apeglm para reducir los coeficientes, el cual es bueno para reducir las estimaciones de LFC que se consideraran ruido al mismo tiempo que proporciona estimaciones de LFC de bajo sesgo para diferencias verdaderamente grandes [5]. Para usar apeglm, especificamos un coeficiente a encoger del modelo entre los que aparecen en resultsNames (dds). En nuestro caso slo tenemos el coeficiente correspondiente a la comparacin sano vs mieloma mltiple:

resultsNames(dds)
[1] "Intercept"                                               
[2] "Sample_source_name_ch1_Multiple_Myeloma_vs_Healthy_Adult"

plotMA(MAdata, main = 'MA plot para comparacin Healthy vs MM', ylim = c(-10,10)) 
topGene <- rownames(MAdata)[which.min(MAdata$padj)]
with(MAdata[topGene, ], {
